1 Introduction to the ASN.1 Library

This section provides a basic introduction to defining ASN.1 types and using the types to encode and decode data using DER.

> (require asn1)

ASN.1 has several familiar base types, such as INTEGER and IA5String. (IA5 is the same as ASCII—the 7-bit character set.)

Racket values are encoded as instances of an ASN.1 type using the DER-encode function, which produces a bytestring (bytes?):

> (DER-encode INTEGER 123456)

#"\2\3\1\342@"

> (DER-encode INTEGER (expt 10 30))

#"\2\r\f\237,\234\320Ft\355\352@\0\0\0"

> (DER-encode IA5String "I am the walrus.")

#"\26\20I am the walrus."

A DER bytestring is decoded according to an ASN.1 type to get a Racket value using the DER-decode function:

> (DER-decode INTEGER (DER-encode INTEGER 123456))

123456

> (DER-decode INTEGER (DER-encode INTEGER (expt 10 30)))

1000000000000000000000000000000

> (DER-decode IA5String (DER-encode IA5String "I am the walrus."))

"I am the walrus."

Complex types are created using forms such as Sequence, Choice, and SequenceOf. For example, here is an ASN.1 type for a sequence of integers:

> (define Integers (SequenceOf INTEGER))
> (DER-encode Integers '(1 2 3 -1000))

#"0\r\2\1\1\2\1\2\2\1\3\2\2\374\30"

> (DER-decode Integers (DER-encode Integers '(1 2 3 -1000)))

'(1 2 3 -1000)

Unlike SequenceOf, Sequence and Choice take multiple components, each labeled with a name and optionally a tagging directive.

Here is an ASN.1 type representing a three-dimensional point:

> (define Point (Sequence [x INTEGER] [y INTEGER] [z INTEGER]))
> (DER-encode Point (hasheq 'x 123 'y 456 'z 789))

#"0\v\2\1{\2\2\1\310\2\2\3\25"

And here’s one representing a reference to a person:

> (define Person (Choice [name IA5String] [number INTEGER]))
> (DER-encode Person '(name "Jean"))

#"\26\4Jean"

> (DER-encode Person '(number 24601))

#"\2\2`\31"

> (DER-decode Person (DER-encode Person '(number 24601)))

'(number 24601)

Sometimes components of a Choice (and sometimes other structured types) must be given alternative tags because their default tags would not distinguish between them.

> (define Employee
    (Choice [name  #:implicit 0 IA5String]
            [title #:implicit 1 IA5String]))
> (DER-encode Employee '(name "Ash"))

#"\200\3Ash"

> (DER-encode Employee '(title "Boomstick Specialist"))

#"\201\24Boomstick Specialist"

If an encoded value uses only built-in types without alternative tagging, it can be decoded using the type ANY instead of the specific ASN.1 type to which it belongs. Values cannot be encoded as ANY.

> (DER-decode ANY (DER-encode INTEGER 123456))

123456

> (DER-decode ANY (DER-encode IA5String "I am the walrus."))

"I am the walrus."

Structured values decoded as ANY do not have symbolic labels; those are part of the type, not the encoding. Sequences are decoded as if they were (SequenceOf ANY).

> (DER-decode ANY (DER-encode Point (hasheq 'x 123 'y 456 'z 789)))

'(123 456 789)

> (DER-decode ANY (DER-encode Person '(name "Jean")))

"Jean"

Finally, encodings that use context-specific tags, such as Employee above, cannot be decoded with ANY:

> (DER-decode ANY (DER-encode Employee '(name "Ash")))

DER-decode-value: non-universal tag found decoding ANY

  tag: context-specific 0 primitive
