3 DER Encoding and Decoding

One common encoding (or “transfer syntax”) for ASN.1 types is called DER (for “distinguished encoding rules”). It is a restricted form of BER (“basic encoding rules”).

A DER-encoded value is logically a tag, length, value (TLV) triple. The tag indicates to what type the value belongs (or at least disambiguates; the tag is often context-specific). The tag also indicates whether the encoding is primitive or constructed—that is, consisting of a sequence of TLV triples. The length indicates the length of the value encoding, so even unknown types can be decomposed into TLV triples. The value component contains a type-specific encoding of the value. For example, integers are encoded in big-endian, base-256, two’s-complement form using a minimal number of octets.

procedure

(DER-encode type v) → bytes?

  type : asn1-type?
  v : any/c
Encodes v as ASN.1 type type using DER, producing a TLV triple.

procedure

(DER-decode type b) → any/c

  type : asn1-type?
  b : bytes?
Decodes the DER TLV triple b as ASN.1 type type, producing a Racket value.

procedure

(DER-read type in) → any/c

  type : asn1-type?
  in : input-port?
Reads one DER TLV triple from in and decodes it as type, producing a Racket value.

3.1 Low-level DER Operations

struct

(struct DER-frame (tagclass tagkind tagnum value))

  tagclass : (or/c 'universal 'private 'application 'context-sensitive)
  tagkind : (or/c 'primitive 'constructed)
  tagnum : exact-nonnegative-integer?
  value : bytes?
Represents a decomposed TLV triple. The tag is broken down into the tagclass, tagkind, and tagnum fields. The length is not represented; it is implicit in the value field.

procedure

(DER-frame->bytes frame) → bytes?

  frame : DER-frame?

procedure

(bytes->DER-frame b) → DER-frame?

  b : bytes?

procedure

(read-DER-frame in) → DER-frame?

Encode, decode, or read a DER frame, respectively.

Example:
> (bytes->DER-frame (DER-encode INTEGER 123456))

(DER-frame 'universal 'primitive 2 #"\1\342@")

procedure

(DER-encode-value type v) → bytes?

  type : asn1-type?
  v : any/c
Encodes v as ASN.1 type type using DER, producing a raw value component instead of the full TLV triple.

Examples:
> (DER-encode-value INTEGER 123456)

#"\1\342@"

> (bytes->DER-frame (DER-encode INTEGER 123456))

(DER-frame 'universal 'primitive 2 #"\1\342@")

procedure

(DER-decode-value type b) → any/c

  type : asn1-type?
  b : bytes?
Decodes the raw value component b as ASN.1 type type, producing a Racket value.

3.2 DER Encoding and Decoding Hooks

Encoding and decoding can be modified through type-indexed hooks, which control the encoding and decoding of the value components of the corresponding types. Hooks cannot influence the handling of tag or length components—except in the special case of ANY type.

The hook parameters use eq? for type comparison. Use (Wrap type) to generate a type equivalent to type but distinct for the purpose of targeting hooks.

parameter

(DER-encode-hooks)

 → 
(listof
 (or/c
  (list/c asn1-type? 'pre (-> any/c any/c))
  (list/c asn1-type? 'encode (-> any/c bytes?))))
(DER-encode-hooks hooks) → void?
  hooks : 
(listof
 (or/c
  (list/c asn1-type? 'pre (-> any/c any/c))
  (list/c asn1-type? 'encode (-> any/c bytes?))))
Parameter of hooks for controlling encoding. There are two kinds of hooks, distinguished by whether they are run before or instead of the built-in decoding rules:

  • (list type 'pre pre-encoder) — When encoding a Racket value v as ASN.1 type type, pass the result of (pre-encoder v) to the built-in encoder (or to the next encoder hook) instead of v.

  • (list type 'encode encoder) — When encoding a Racket value v as ASN.1 type type, use the bytestring result of (encoder v) as the value component instead of calling the built-in encoder rules.

Encoding hooks can be used for efficiency. Here’s an alternative to the example discussed in the Wrap form documentation:

> (define sig
    (parameterize ((DER-encode-hooks
                    (list (list INTEGER 'encode (lambda (b) b)))))
      (DER-encode (Sequence [r INTEGER] [s INTEGER])
                  (hasheq 'r #"}nSi|-uy"
                          's #"y\21~P#3\37\b"))))
> sig

#"0\24\2\b}nSi|-uy\2\by\21~P#3\37\b"

> (DER-decode (Sequence [r INTEGER] [s INTEGER]) sig)

'#hasheq((s . 8723892835846397704) (r . 9038253214699189625))

As with Wrap, no checking is done on the returned bytestring.

parameter

(DER-decode-hooks)

 → 
(listof
 (or/c
  (list/c asn1-type? 'decode (-> bytes? any/c))
  (list/c asn1-type? 'post (-> any/c any/c))))
(DER-decode-hooks hooks) → void?
  hooks : 
(listof
 (or/c
  (list/c asn1-type? 'decode (-> bytes? any/c))
  (list/c asn1-type? 'post (-> any/c any/c))))
Parameter of hooks for controlling decoding. There are two kinds of hooks, distinguished by whether they are run before or after the built-in decoding rules:

  • (list type 'decode decoder) — When decoding a bytestring b (the value component of a DER encoding) as ASN.1 type type, use the result of (decoder b) instead of the built-in decoding rules for that type.

  • (list type 'post post-decoder) — is interpreted as follows: After decoding a bytestring as ASN.1 type type to a Racket value v using the built-in decoding rules (or a decoder hook), evaluate (post-decoder v) and return the result as the decoded value.
