5 ASN.1-Related Utilities

5.1 Base-256 Integer Representations

 (require asn1/base256) package: asn1-lib

procedure

(signed->base256 n) → bytes?

  n : exact-integer?

procedure

(base256->signed b) → exact-integer?

  b : bytes?
Converts between integers and their big-endian, base-256, two’s complement signed encodings—the encoding used by the ASN.1 INTEGER type.

Examples:

> (signed->base256 1)

#"\1"

> (signed->base256 255)

#"\0\377"

> (signed->base256 256)

#"\1\0"

> (signed->base256 -1)

#"\377"

> (signed->base256 (expt 10 40))

#"\35c)\361\303\\\244\277\253\271\365a\0\0\0\0\0"

> (base256->signed (bytes 127))

127

> (base256->signed (bytes 128))

-128

> (base256->signed (bytes 1 1))

257

procedure

(unsigned->base256 n) → bytes?

  n : exact-nonnegative-integer?

procedure

(base256->unsigned b) → exact-nonnegative-integer?

  b : bytes?
Converts between nonnegative integers and their big-endian base-256 unsigned encodings.

Examples:

> (unsigned->base256 1)

#"\1"

> (unsigned->base256 255)

#"\377"

> (unsigned->base256 256)

#"\1\0"

> (unsigned->base256 (expt 10 40))

#"\35c)\361\303\\\244\277\253\271\365a\0\0\0\0\0"

> (base256->unsigned (bytes 127))

127

> (base256->unsigned (bytes 128))

128

> (base256->unsigned (bytes 1 1))

257

procedure

(base256-normalize-signed b) → bytes?

  b : bytes?

procedure

(base256-normalize-unsigned b) → bytes?

  b : bytes?
Normalizes the given big-endian base-256 signed or unsigned (respectively) encoding. Unnecessary leading 0 bytes are removed from nonnegative integers, and unnecessary leading 255 (-1) bytes are removed from negative integers.

Equivalent to (signed->base256 (base256->signed b)) and (unsigned->base256 (base256->unsigned b)), respectively, but faster.

Examples:

; signed
> (base256-normalize-signed (bytes 0 0 0 1))

#"\1"

> (base256-normalize-signed (bytes 0 128 0 1))

#"\0\200\0\1"

> (base256-normalize-signed (bytes 255 255 128))

#"\200"

> (base256-normalize-signed (bytes 255 255 1))

#"\377\1"

; unsigned
> (base256-normalize-unsigned (bytes 0 0 0 1))

#"\1"

> (base256-normalize-unsigned (bytes 0 128 0 1))

#"\200\0\1"

> (base256-normalize-unsigned (bytes 255 255 1))

#"\377\377\1"

procedure

(base256-unsigned->signed b) → bytes?

  b : bytes?
Converts the unsigned encoding of a number to the (normalized) signed encoding—essentially, adds a leading 0 byte when necessary.

Equivalent to (signed->base256 (base256->unsigned b)), but faster.

Examples:

> (base256-unsigned->signed (bytes 127))

#"\177"

> (base256-unsigned->signed (bytes 128))

#"\0\200"

procedure

(base256-signed-zero? b) → boolean?

  b : bytes?

procedure

(base256-signed-positive? b) → boolean?

  b : bytes?

procedure

(base256-signed-negative? b) → boolean?

  b : bytes?
Determines whether b is the signed encoding of zero, a positive integer, or a negative integer, respectively.

Equivalent to (zero? (base256->signed b)), (positive? (base256->signed b)), and (negative? (base256->signed b)), respectively.
